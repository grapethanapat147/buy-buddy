## 1. Concept — แก่นของ product

แอปช่วย **วางแผนและแนะนำการซื้อของ** สำหรับคนเช่าห้อง — **เป็นตัวช่วยตัดสินใจ ไม่ใช่ร้านค้า/ไม่ใช่ e-commerce ที่ขายเอง** ตัวเว็บจบหน้าที่ตรงที่ "ทำให้ผู้ใช้ตัดสินใจได้ง่ายขึ้น" แล้วส่งไปซื้อจริงที่ supermarket / e-commerce ภายนอก

ฟีลการใช้งาน: **"จัดกระเป๋า/จัดลิสต์ก่อน แล้วค่อยไปซื้อจริง"**

ประโยคเดียว:
> "แอปที่ช่วยคนเช่าห้องวางแผนว่าต้องซื้ออะไรเข้าห้องบ้างตามงบ พร้อมเทียบราคาข้ามร้านให้ แล้วส่งไปซื้อจริงข้างนอก"

### กลุ่มเป้าหมาย
นักศึกษาจบใหม่ / คนเริ่มทำงานที่เพิ่งเช่าห้อง ยังไม่รู้ว่า "ต้องซื้ออะไรเข้าห้องบ้าง" และ "อันไหนดี/คุ้มตามงบ" — มักอยู่คนเดียวครั้งแรก งบจำกัด กลัวซื้อผิด/ซื้อเกิน

### 2 โหมดการใช้งาน
- **Move-in** — setup ห้องใหม่ (ของถาวร เช่น หม้อหุงข้าว พัดลม ที่นอน) ทำครั้งเดียว
- **Restock** — ของสิ้นเปลือง (ข้าวสาร ผงซักฟอก ทิชชู่) วนซ้ำรายสัปดาห์/เดือน/ปี

### คุณค่า 4 ชั้น
1. **Spec-based Recommendation** — รับโจทย์ผู้ใช้ (งบ/ห้อง/จำนวนคน/ทำอาหารไหม) แล้วแนะนำของที่ควรมี
2. **Cascading "ซื้อคู่กัน" (เป็นทอดๆ)** — แนะนำของที่ต้องใช้ร่วมกันต่อเนื่อง (หม้อหุงข้าว → ข้าวสาร → กล่องเก็บ)
3. **Price Aggregation** — หาราคาคุ้มสุดข้าม platform
4. **Planning** — จัดของเป็นลิสต์ตามช่วงเวลา (สัปดาห์/เดือน/ปี)

## 2. Sitemap

```
🏠 GROCERY LIST WEB APP
│
├── A. ทางเข้า (Guest เข้าได้หมด) — 2 ประตู
│   ├── 1. Landing / Home ── ขายคุณค่า + [เริ่มด้วย Wizard] / [เลือกดูของเอง]
│   ├── 2. Onboarding Wizard ── ป้อน spec ทีละสเต็ป → เจนชุดของแนะนำ
│   ├── 3. Explore / Browse ── เดินดูของเองตามหมวด/งบ (ประตูที่ 2)

│
├── B. แกนหลัก (loop ที่วนซ้ำ — Guest ใช้ได้ / เซฟต้องสมัคร)
│   ├── 4. ชุดของแนะนำ (Recommendations) ── Toggle Move-in|Restock + filter
│   ├── 5. รายละเอียดสินค้า (Product Detail) ── เทียบราคา + Smart Bundle
│   └── 6. แผนของฉัน (My Plan) ── หน้าเดียว 2 แท็บ (รายการ / ปฏิทิน)
│
└── C. บัญชีผู้ใช้ (ปลดล็อกตอนเซฟ)
```

## 4. Page-level Blueprint

### 1. Landing — *เข้าใจใน 5 วิ + เลือกประตู*
- Hero: 1 ประโยคคุณค่า + ภาพ "จัดของเข้าห้อง"
- 2 ปุ่ม: `[เริ่มด้วย Wizard]` (เด่นสุด) · `[เลือกดูของเอง]`
- **Sample kit preview** — การ์ดตัวอย่าง "ชุดเริ่มต้นห้องสตูดิโอ ฿4,900" (เห็นของจริงก่อนตัดสินใจ)
- How it works 3 สเต็ป (ตอบ → แนะนำ → ไปซื้อจริง)

### 2. Onboarding Wizard — *เก็บ spec โดยไม่เหมือนกรอกฟอร์ม*
- Progress bar (1/5) + ปุ่ม `ข้ามไป Browse`
- คำถามทีละข้อ แบบการ์ด/ปุ่มใหญ่: งบ (slider) · ประเภทห้อง · อยู่กี่คน · ทำอาหารไหม · มีของอะไรอยู่แล้ว (checklist)
- **Live preview**: จำนวนของ + ยอดรวมขยับตามที่ตอบ
- จบ: `[ดูชุดของแนะนำ]`

### 3. Explore / Browse — *ทางเข้าคนที่รู้ว่าจะซื้ออะไร*
- Search + filter (หมวด / งบ / must vs nice-to-have)
- Grid การ์ดสินค้า + ปุ่ม `+ ใส่ลงแผน` บนการ์ด
- Sort: ราคา / คุ้มค่า / ยอดนิยม

### 4. ชุดของแนะนำ (Recommendations) — *หัวใจ loop*
- Toggle: `[ Move-in ]` | `[ Restock ]`
- หมวด collapsible: ครัว · นอน · ห้องน้ำ · ทำความสะอาด · ของกินตุน
- การ์ดสินค้า: รูป · ชื่อ · ราคาคุ้มสุด · badge `จำเป็น`/`ถ้ามีงบ` · `+ ใส่ลงแผน`
- Filter bar + สรุปยอดรวมเทียบงบ (ค้างบน)

### 5. รายละเอียดสินค้า (Product Detail)
- แกลเลอรี + ชื่อ + สเปคย่อ
- **บล็อกเทียบราคา**: default โชว์ร้านคุ้มสุด 1 อัน + `ดูอีก N ร้าน` (พับไว้) + ป้าย **"ราคาอ้างอิง · อัปเดต [วันที่]"**
- **"ทำไมแนะนำ"**: เหตุผล 1 บรรทัด
- **Smart Bundle "มักซื้อคู่กับ"**: 3–4 ชิ้น + `ใส่ทั้งชุด`
- ปุ่มคู่: `+ ใส่ลงแผน` · `ไปซื้อจริง ↗` (tag เปิดเผย affiliate)

### 6. แผนของฉัน (My Plan) — *2 แท็บ ข้อมูลชุดเดียว*
- แท็บ **รายการ**: ลิสต์ Move-in / Restock · checkbox "จัดแล้ว" · **progress 8/12** · สรุปงบรวม + ส่วนต่าง · มุมมอง **"ยอดรวมต่อร้าน"**
- แท็บ **ปฏิทิน**: วาง Restock ราย สัปดาห์/เดือน + ตั้งเตือนรอบถัดไป
- ปุ่ม `เซฟแผน` (เด้ง signup ถ้า guest) · `ส่งออกลิสต์`

### 7. สมัคร / ล็อกอิน (overlay)
- Modal ทับ ไม่ใช่หน้าใหม่ · ข้อความ "เซฟแผนของคุณไว้ ไม่ให้หาย"
- Social login + email · มี `ข้ามไปก่อน`
- **Lossless**: สมัครเสร็จอยู่หน้าเดิม แผนไม่หาย

### 8. โปรไฟล์ / Account
- แผนที่เซฟ · แก้ spec/งบ (recommendation อัปเดตตาม) · ตั้งค่าการแจ้งเตือน

## 5. User Flow — persona "ฟ้า"

**ฟ้า** จบใหม่ อายุ 22 งานแรก เพิ่งเช่าสตูดิโอ **งบ 5,000 บาท** ไม่เคยอยู่คนเดียว กลัวซื้อผิด/เกิน

| # | จอ | ฟ้าทำ | ระบบตอบ | อารมณ์/หลักการ |
|---|---|---|---|---|
| 1 | Landing | เปิดครั้งแรก เห็น "ชุดเริ่มต้น ฿4,900" | โชว์ของจริงทันที | เห็นคุณค่าก่อน |
| 2 | Landing | กด `เริ่มด้วย Wizard` | เข้า wizard 1/5 | ไม่กดดัน มีทางออก |
| 3 | Wizard | งบ 5,000 · สตูดิโอ · คนเดียว · ทำอาหารบ้าง | preview "18 ชิ้น ฿4,650" | feedback สด |
| 4 | Recommendations | ดูของแยกหมวด | Budget meter ฿4,650/5,000 | สบายใจ อยู่ในงบ |
| 5 | Product Detail | ดูหม้อหุงข้าว | คุ้มสุด ฿590 + เหตุผล | ไม่ต้องเทียบเอง |
| 6 | Smart Bundle | เห็น "ซื้อคู่กับ ข้าวสาร+ทัพพี+กล่องข้าว" | กด `ใส่ทั้งชุด` | นึกไม่ถึง! |
| 7 | Budget meter | ยอด → ฿5,120 **เกินงบ เมตรแดง** | เตือน + แนะตัดของ | เตือนอย่างเป็นมิตร |
| 8 | My Plan | ย้าย 2 ชิ้นไป Restock | ยอด ฿4,780 เมตรเขียว | ควบคุมได้เอง |
| 9 | Save | กด `เซฟแผน` | overlay signup | สมัครเพราะเห็นเหตุผล |
| 10 | Signup | ล็อกอิน Google | กลับหน้าเดิม แผนครบ | lossless |
| 11 | ปฏิทิน | ตั้งเตือน "ทุกอาทิตย์" | ยืนยัน notification | loop กลับมา |
| 12 | จบ | `ส่งออกลิสต์` ไปซื้อจริง | เปิดลิสต์ + ลิงก์ร้าน | แอปจบหน้าที่ |

**จุด make-or-break:** สเต็ป 7–8 (เกินงบ) — จัดการดี = "แอปเข้าใจฉัน"; จัดการห่วย = โดนดุแล้วปิดเว็บ

## 6. UX Principles — 6 moves

| # | ปัญหา | Move | หลักการ |
|---|---|---|---|
| 1 | Wizard บังคับตอบก่อนเห็นคุณค่า | แอบดูชุดของตัวอย่างก่อนตอบครบ | Time-to-value |
| 2 | "งบ" ไม่มีตัวตนบนจอ | Budget meter ติดขอบจอตลอด | Feedback / ลดความกังวล |
| 3 | จุดต่าง (ซื้อคู่กัน) ถูกฝัง | ยกเป็น "Smart Bundle" ให้เห็นชัด | Discoverability |
| 4 | เทียบราคา = decision fatigue | default "คุ้มสุด" + "ทำไมแนะนำ" | ลด cognitive load / trust |
| 5 | Account เป็นกำแพง | guest→account ไม่สูญข้อมูล | Lossless onboarding |
| 6 | Planner เป็นแค่ที่เก็บของ | checklist progress + เตือน restock | Retention |

> ของพวกนี้คือ **สมมติฐานเรื่องคุณค่า** ควร validate กับผู้ใช้จริงก่อนลงมือ

## 7. Key Decisions — 3 ข้อวิกฤต (ตัดสินแล้ว)

### ข้อ 1 — แหล่งข้อมูลราคา
**✅ Curated reference-price DB (แอดมิน seed ผ่าน Filament) → affiliate API ทีหลัง**
- ทำ: เก็บ "ราคาอ้างอิง" ในฐานข้อมูลเรา ติดป้าย *"ราคาอ้างอิง · อัปเดต [วันที่]"* ราคาจริงคอนเฟิร์มบน platform ตอนกดลิงก์
- ❌ ไม่ทำ: scraping real-time (ผิด ToS + เปราะบาง) — **ห้ามใน MVP**
- ทำไม: สมมติฐานที่ต้อง validate คือ "คนอยากได้ตัวช่วยจัดของตามงบ" ไม่ใช่ "ราคาถูกสุด real-time" — ตรง concept "advisor"
- Phase 2: เสียบ Shopee/Lazada affiliate API เฉพาะที่มี

### ข้อ 2 — เทียบราคา ชิ้น vs ตะกร้า
**✅ MVP = ถูกสุดรายชิ้น + สรุปยอดรวมต่อร้าน (ไม่ใช่ optimizer เต็ม)**
- ทำ: หน้าสินค้าโชว์คุ้มสุดรายชิ้น + My Plan โชว์ "ถ้าซื้อทั้งหมดที่ร้านนี้ = ฿X" สำหรับ 1–2 platform
- ❌ ไม่ทำ (ตอนนี้): basket optimizer เต็ม (combination + free-shipping threshold + ค่าส่ง)
- ทำไม: ตอบคำว่า "รวมกัน" แบบค่อยเป็นค่อยไป ไม่หลอกว่าฉลาดกว่าที่เป็น
- Phase 2: optimizer จริง (รวมค่าส่ง + เงื่อนไข "ครบ ฿X ฟรีส่ง")

### ข้อ 3 — โมเดลรายได้
**✅ Affiliate commission เป็นหลัก + แอปฟรี + กฎเหล็กความโปร่งใส**
- ทำ: รายได้จาก affiliate ผ่านลิงก์ "ไปซื้อจริง" (ผู้ใช้ไม่จ่าย)
- กฎเหล็ก: การจัดอันดับ "คุ้มสุด" มาจากราคา/ความคุ้มค่าจริงเท่านั้น **ห้าม pay-to-rank** + **เปิดเผยว่าเป็น affiliate**
- ❌ ไม่ทำ: โฆษณา banner · subscription ตั้งแต่แรก
- ทำไม: โมเดลเดียวที่ไม่ขัด concept "advisor" — แต่มี incentive conflict ในตัว กฎเหล็กคือสิ่งที่กันไม่ให้ทำลาย trust
- Phase 2: premium ทางเลือก (แจ้งเตือนราคาลด, วางแผนขั้นสูง)

### ชุดที่ประกอบกัน
> ราคาอ้างอิงในฐานข้อมูลเรา → โชว์คุ้มสุดรายชิ้น + ยอดรวมต่อร้าน → กดลิงก์ affiliate ไปคอนเฟิร์มราคาจริงและซื้อ

**ผลกระทบต่อดีไซน์:**
1. Product Detail ต้องมีป้าย "ราคาอ้างอิง · อัปเดต [วันที่]" + tag เปิดเผย affiliate
2. My Plan เพิ่มมุมมอง "ยอดรวมต่อร้าน"
3. เพิ่มงาน backend: Filament admin จัดการ catalog + ราคา (เข้า Must)

## 8. MoSCoW — Scope MVP

เป้า MVP: **พิสูจน์ว่า "แนะนำของตามงบ + เทียบราคา" มีคุณค่าจริง** ด้วย effort น้อยสุด

**🔴 Must**
- Wizard เก็บ spec (งบ/ห้อง/คน/ทำอาหาร)
- Recommendations แยกหมวด + Budget meter
- Product Detail + เทียบราคา (curated reference price)
- My Plan แท็บรายการ + สรุปงบ + ยอดรวมต่อร้าน
- ส่งออก/ลิงก์ affiliate ไปซื้อจริง
- **Filament admin จัดการ catalog + ราคา** (เพิ่มจากข้อ 1)

**🟡 Should**
- Smart Bundle "ซื้อคู่กัน"
- Guest→account แบบ lossless
- "ทำไมแนะนำ"
- Explore/Browse

**🟢 Could**
- แท็บปฏิทิน + แจ้งเตือน Restock
- Sample kit preview บน Landing
- Progress/celebration

**⚪️ Won't (MVP)**
- ระบบจ่ายเงิน/ตะกร้าซื้อจริง (แอปเป็นแค่ตัวแนะนำ)
- Basket optimizer เต็ม + API ราคา real-time
- ระบบ social/แชร์แผน
- แอป mobile native
